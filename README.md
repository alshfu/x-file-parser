# x-file-parser

Парсер и локальная читалка манги/комиксов на чистом Node.js — без npm-зависимостей.

**Pipeline:** качаем главы → раскладываем по томам → читаем в браузере через локальный сервер.

```
┌──────────────┐     ┌──────────────┐     ┌──────────────────┐
│  index.js    │ ──▶ │ downloads/   │ ──▶ │ reader-server.js │
│  (парсер)    │     │ <Серия>/...  │     │  + reader.html   │
└──────────────┘     └──────────────┘     └──────────────────┘
```

## Возможности

- Скачивание всей серии или произвольного диапазона глав по URL.
- Автоматический обход PoW-челленджа сайта.
- Идемпотентный manifest: повторный запуск докачивает то, чего не хватает.
- Локальная SPA-читалка: автоматический разворот по соотношению сторон, клавиатура/тач/свайп, прогресс по томам, продолжение чтения, изображение во весь экран по вертикали.

## Требования

- Node.js ≥ 20 (нужны встроенный `fetch` и `Headers.getSetCookie`).
- Никаких npm-зависимостей.

## Запуск

```bash
# скачать серию (URL главы или серии)
npm run parse -- https://example.com/reader/<news_id>/<chapter_id>

# или несколько за раз
npm run parse -- <url1> <url2> <url3>

# поднять читалку
npm run serve
# → http://localhost:3000
```

## Структура

```
.
├── index.js                 ← парсер (основной)
```

## index.js — парсер

URL может быть двух видов:
- **Страница главы** (`/reader/<news_id>/<chapter_id>`) — качает с этой главы и всё, что после.
- **Страница серии** (без `/reader/...`) — сначала резолвится в первую главу, потом качается **вся серия** с начала.

### Что делает парсер пошагово

1. **Обход анти-бот защиты.** Сайт отдаёт 302 → `/_c?...` (challenge с PoW). Парсер:
   - получает токен (`token: "..."` в HTML челленджа);
   - решает SHA-256 PoW (ищет nonce такой, чтобы `sha256(token + ':' + nonce)` начинался с `00`);
   - имитирует «человеческое» время решения (450–700 мс);
   - POST на `/_v` с метриками (screen, GPU, timezone, …) → получает cookie доступа.
2. **Скачивание страницы главы.** Извлекается `window.__DATA__ = {...};` — оттуда:
   - `news_id`, `chapter_id`;
   - `chapters[]` — полный каталог серии (id + title);
   - `images[]` — относительные пути к картинкам текущей главы;
   - `host` — обычно `img.<домен>`.
3. **Имя серии** берётся из `<title>Читать <name> — ...`.
4. **Очередь глав.** Реверс `chapters` (на сайте они идут от новых к старым). Если стартовая глава — внутри списка, режется до неё; если URL был «серийный», берётся всё.
5. **Том.** Извлекается из title главы: `/Том\s+(\d+)/i`, иначе `1`. Главы с пометками «альт» / `alt` (альтернативные переводы) **пропускаются**.
6. **Скачивание картинок.** Сохраняются как `том N/page <N>.<ext>` со сквозной нумерацией в пределах тома.
7. **Манифест** `<серия>/.manifest.json` — для дозагрузки и идемпотентного перезапуска:
   ```json
   {
     "title": "...",
     "news_id": 3038,
     "completed": { "<chapter_id>": { "volume": 1, "pages": 19, "title": "..." } },
     "counters":  { "1": 19, "2": 38 }
   }
   ```
   Главы из `completed` повторно не качаются. `counters[vol]` — последняя использованная страница в томе.
8. **Ретраи** (`guardedFetch`):
   - 302 → `/_c?` → проходит челлендж и повторяет;
   - 302 на обычный URL → следует;
   - 403 / 429 → экспоненциальная пауза (2s × 2^attempt);
   - 404 → 2 коротких ретрая;
   - сетевые ошибки → 1.5s × 2^attempt, 5 попыток;
   - между картинками — `sleep 150 ms`, между главами — `sleep 800 ms`.

### Где может сломаться

- Регэксп `chapter_id` берёт **первое** вхождение `/reader/\d+/\d+` со страницы серии — обычно это последняя/новейшая глава.
- `extractSeriesName` падает если меняется паттерн `<title>Читать ... —`.
- Сложность PoW зашита (`startsWith('00')` — 8 ведущих бит). Если сайт поднимет до `000`, нужно поправить.

## bootstrap-manifest.mjs

Одноразовая утилита: была написана, когда часть томов уже лежала в `downloads/`, но без `.manifest.json`. Скрипт:
- скачивает первую главу;
- помечает все главы уже скачанных томов как `completed` (с `pages: -1` — маркер «скачано вне парсера»);
- считает фактические `counters` по файлам на диске;
- пишет `.manifest.json`.

В обычной работе **не нужен**.

## reader-server.js + reader.html — читалка

`npm run serve` → http://localhost:3000

### API (читает из `./downloads`)

| Endpoint                                | Ответ                                       |
| --------------------------------------- | ------------------------------------------- |
| `GET /api/series`                       | `[{name, volumes, pages}]`                  |
| `GET /api/series/<name>`                | `{volumes: [{name, volume, pages}]}`        |
| `GET /api/series/<name>/<volume>`       | `{pages: [filename]}`                       |
| `GET /files/<series>/<volume>/<file>`   | байты картинки (safe-join, кэш 24ч)         |
| `GET /`                                 | `reader.html`                               |

### Фронтенд

`reader.html` — одностраничный SPA, три view (`series` → `volumes` → `reader`), история через `history.pushState` (`?s=…&v=…&p=…`).

- **Авто-разворот**: если первая страница тома по соотношению сторон портретная (`H > W*1.05`) — режим парного разворота, иначе одиночный.
- **Управление**: стрелки / `Space` / `Home` / `End`, клик по левой и правой трети экрана, тач-свайп; центральный тап — скрыть UI; кнопка «Перейти» → промпт с номером страницы.
- **Прогресс**: сохраняется в `localStorage` (per-volume + per-series), карточка «Продолжить чтение» на главной.
- **Предзагрузка**: 3 следующих разворота в обе стороны.
- **Анимация перелистывания**: 3D rotate + drop-shadow.

## Зависимости

Чистый Node.js (ESM, `type: module`). Никаких npm-пакетов — только встроенные модули (`node:fs`, `node:crypto`, `node:http`, `fetch`).
